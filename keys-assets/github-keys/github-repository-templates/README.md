# GitHub Keys: Repository Templates

**Version**: 1.0.0  
**Tool**: GitHub  
**Maturity**: Starter  
**Outcome**: Automation

---

## What This Key Unlocks

Pre-configured repository structures for common project types. This key unlocks:

- **SaaS Starter**: Complete SaaS application template
- **API Starter**: REST API template
- **CLI Starter**: Command-line tool template
- **Full-Stack Starter**: Full-stack application template
- **Library Starter**: Open-source library template

---

## Installation

1. **Use GitHub template**:
   - Click "Use this template" on any template repository
   - Create your new repository from the template

2. **Or clone and customize**:
   ```bash
   git clone <template-repo-url>
   cd <template-name>
   # Customize for your project
   ```

## Included Templates

### SaaS Starter
Complete SaaS application with frontend, backend, database, auth, and billing.

**Includes**:
- Next.js frontend
- Express.js backend
- Supabase database
- Stripe billing
- GitHub Actions CI/CD

### API Starter
REST API template with validation, error handling, and documentation.

**Includes**:
- Express.js/Node.js
- TypeScript
- Zod validation
- OpenAPI documentation
- Testing framework

### CLI Starter
Command-line tool template with argument parsing and help system.

**Includes**:
- Node.js CLI framework
- Argument parsing (commander.js)
- Help system
- Testing

### Full-Stack Starter
Full-stack application with frontend and backend.

**Includes**:
- React/Next.js frontend
- Express.js backend
- Shared types
- Development setup

### Library Starter
Open-source library template with build, test, and publish setup.

**Includes**:
- TypeScript library structure
- Build configuration
- Testing setup
- Publishing workflow

## Usage

### Using a Template

1. **Choose a template** from the included templates
2. **Click "Use this template"** on GitHub
3. **Create your repository** from the template
4. **Follow setup instructions** in the template's README

### Customizing Templates

All templates are starting points. Customize:

- Project name and description
- Dependencies and versions
- Configuration files
- Documentation
