# AI Output Regression Incident Runbook - Checklist

## Immediate Safety Checks

- [ ] Check AI service status
- [ ] Check recent model version changes
- [ ] Assess user impact

## Diagnosis Flow

- [ ] Step 1: Identify regression pattern (use Jupyter KEY)
- [ ] Step 2: Check model version
- [ ] Step 3: Check input quality (use Jupyter KEY)

## Action Steps

- [ ] Action 1: Rollback model version (if version change)
- [ ] Action 2: Adjust AI parameters (if parameter issue)
- [ ] Action 3: Enable output validation (if needed)

## Verification

- [ ] Immediate: Test AI outputs (5 minutes)
- [ ] Immediate: Monitor quality scores (5 minutes)
- [ ] Ongoing: Monitor user reports (24 hours)
- [ ] Ongoing: Monitor quality metrics (24 hours)

## Evidence & Audit

- [ ] Record regression start time
- [ ] Record regression pattern
- [ ] Record root cause
- [ ] Record actions taken
- [ ] Save AI output samples
- [ ] Save Jupyter KEY outputs

## Post-Incident

- [ ] Review AI configuration (24 hours)
- [ ] Update monitoring (24 hours)
- [ ] Implement preventative measures (1 week)
