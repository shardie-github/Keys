## Overview

This directory contains all KEYS demo and pitch artifacts. These documents are **canonical**—they govern all sales conversations, investor presentations, and partner discussions.

**Brand Rules (IMMUTABLE):**
- KEYS is the product
- A Key is a discrete asset
- Keyring is the user's collection
- KEYS never replaces tools
- KEYS never runs customer systems

If language contradicts this, it must be corrected.

## Usage Guidelines

### For Sales Teams
1. **Start with:** Elevator Pitch (30s/60s)
2. **Demo with:** 3-Minute Demo Script or 8-Minute Enterprise Demo
3. **Handle objections with:** Hostile Demo Responses
4. **Close with:** Canonical Closing Statement
5. **Reference:** FAQ for common questions

### For Founders
1. **Investor meetings:** Investor Pitch Deck Narrative
2. **Partner discussions:** Partner Pitch
3. **Media interviews:** Press Release Template + Elevator Pitch
4. **Launch preparation:** Launch Checklist

### For Marketing Teams
1. **Content creation:** Use brand language from Elevator Pitch
2. **Press releases:** Press Release Template
3. **Website copy:** Reference FAQ and Elevator Pitch
4. **Social media:** Use key messages from Elevator Pitch

## Brand Consistency

### Language Rules
- ✅ Use: "KEYS is the keyring to modern tools"
- ✅ Use: "Proven patterns"
- ✅ Use: "Unlock capability"
- ✅ Use: "Tools you already own"
- ❌ Avoid: "AI-powered" or "AI magic"
- ❌ Avoid: "Revolutionary" or "game-changing"
- ❌ Avoid: "Replaces [Tool]"
- ❌ Avoid: "Auto-pilot" or "automatic"

### Positioning Rules
- **KEYS unlocks tools, doesn't replace them**
- **KEYS provides proven patterns, not experimental code**
- **KEYS is tool-agnostic, outcome-driven**
- **KEYS saves time, reduces cost, lowers risk**

## Maintenance

### When to Update
- **New asset types:** Update demo scripts to include new Key types
- **New buyer personas:** Add variations to Elevator Pitch
- **New pricing tiers:** Update FAQ and demo scripts
- **New discovery logic:** Update demo scripts to show new discovery features

### How to Update
1. **Check brand consistency:** Ensure language follows brand rules
2. **Update all references:** Update all documents that reference changed features
3. **Test language:** Read aloud to ensure it works when spoken
4. **Version control:** Update version numbers and dates
