# Node / Next.js KEYS Cohesion with Jupyter KEYS

**Version**: 1.0.0  
**Last Updated**: 2024-12-30  
**Status**: Canonical — Ensures product consistency across KEY types  
**Purpose**: Maintain unified KEYS identity and user experience

---

## Core Principle

**Node / Next.js KEYS and Jupyter KEYS are the same product—different tools, same philosophy.**

They share naming, tone, trust level, and product identity.

---

## Shared Philosophy

### 1. Unlock Capability, Don't Replace Tools

**Jupyter KEYS**: Unlock outcomes in Jupyter Notebooks  
**Node/Next KEYS**: Unlock capabilities in Node.js/Next.js apps

**Both**: Amplify tools, never compete with them.

---

### 2. Structured, Validated, Reusable Assets

**Jupyter KEYS**: Structured `.ipynb` files with clear outcomes  
**Node/Next KEYS**: Structured TypeScript modules with clear capabilities

**Both**: Reusable, inspectable, deterministic.

---

### 3. Tool-Agnostic, Outcome-Driven

**Jupyter KEYS**: Work across Jupyter environments (local, Colab, cloud)  
**Node/Next KEYS**: Work across Node/Next environments (local, Vercel, cloud)

**Both**: Tool-independent, outcome-focused.

---

### 4. Explicit, Never Magic

**Jupyter KEYS**: Clear cells, visible logic, documented steps  
**Node/Next KEYS**: Explicit imports, visible code, documented integration

**Both**: No hidden behavior, no magic.

---

### 5. Host Sovereignty

**Jupyter KEYS**: Notebooks remain user-owned, KEYS provide patterns  
**Node/Next KEYS**: Apps remain user-owned, KEYS provide capabilities

**Both**: User controls the tool, KEY provides leverage.

---

## Shared Naming Conventions

### KEY Naming

**Jupyter KEYS**: `jupyter-<outcome>-<pattern>`  
**Node/Next KEYS**: `node-<capability>-<pattern>` or `next-<capability>-<pattern>`

**Examples**:
- Jupyter: `jupyter-data-validation-harness`
- Node: `node-stripe-webhook-entitlement`
- Next: `next-audit-log-capture`

---

### Title Format

**Jupyter KEYS**: "Jupyter Keys: [Outcome]"  
**Node/Next KEYS**: "Node Keys: [Capability]" or "Next Keys: [Capability]"

**Examples**:
- Jupyter: "Jupyter Keys: Data Validation Harness"
- Node: "Node Keys: Stripe Webhook Entitlement"
- Next: "Next Keys: Audit Log Capture"

---

### Description Format

**Jupyter KEYS**: "Unlocks [outcome] workflows in Jupyter"  
**Node/Next KEYS**: "Unlocks [capability] in Node.js/Next.js"

**Examples**:
- Jupyter: "Unlocks data validation patterns for ML models in Jupyter"
- Node: "Unlocks Stripe webhook verification and entitlement checking in Node.js"
- Next: "Unlocks audit log dashboard widget in Next.js"
