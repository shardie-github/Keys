# Prompt Review Checklist

Use this checklist before sharing or reusing a prompt.

## Scope and intent

- What task does the prompt cover?
- Who is the intended user?
- What assumptions does the prompt make about tools, data, or access?

## Inputs

- Are required inputs explicit and unambiguous?
- Are sensitive inputs clearly labeled?
- Are defaults safe and reversible?

## Outputs

- Are expected outputs defined?
- Does the prompt instruct the user to validate results?
- Are error conditions or edge cases noted?

## Safety and compliance

- Does the prompt avoid sensitive data or access it with clear consent?
- Are security constraints listed (least privilege, read-only, etc.)?
- Is there a requirement to review before acting?

## Maintenance

- Does the prompt include a version note or owner?
- Is there a path to update or deprecate it?
