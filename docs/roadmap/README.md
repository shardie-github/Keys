## For Future Agents

When developing new keys:

1. **Start with `NEW_KEYS_ROADMAP.md`**
   - Review priority levels
   - Understand context and requirements
   - Check dependencies

2. **Follow Development Process**
   - Planning → Development → QA → Integration → Launch

3. **Update Roadmap**
   - Mark keys as complete
   - Record metrics
   - Update priorities based on learnings

4. **Consider Principles**
   - Keys never compete with tools
   - Keys never hide execution paths
   - Keys always produce tangible outputs
   - Keys must be reusable
