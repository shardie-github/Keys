2. **Development** (varies): Create assets, write tests, document

## Quick Decision Matrix

**When prioritizing a new key, evaluate:**

1. **Market Demand** (40%): User requests, competitive analysis, revenue potential
2. **Tool Coverage** (25%): Ecosystem completeness, maturity gaps, cross-tool synergy
3. **Outcome Coverage** (20%): Category completeness, high-value outcomes, common use cases
4. **Development Feasibility** (15%): Technical complexity, resources, dependencies
